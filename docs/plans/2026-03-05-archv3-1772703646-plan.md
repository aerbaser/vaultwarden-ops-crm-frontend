# Vault Ops CRM Frontend MVP Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a production-ready MVP frontend in `/home/aiadmin/vault-crm-frontend` with the six required routes, secure Vaultwarden-compatible login/session handling, route-level unit coverage, and one deterministic Playwright login/create/share flow.

**Architecture:** Build a Next.js 14 App Router client where route pages are thin composition shells and all behavior is concentrated in typed API endpoints plus Zustand stores. Keep security-critical concerns centralized: cookie-authenticated fetch with CSRF header injection, normalized API errors, centralized auth-failure handling, and memory-only key lifecycle. Implement only the six-screen MVP while preserving extension seams for non-MVP domains.

**Tech Stack:** Next.js 14 (App Router), TypeScript, Zustand, CSS custom properties, Vitest + React Testing Library, MSW, Playwright.
